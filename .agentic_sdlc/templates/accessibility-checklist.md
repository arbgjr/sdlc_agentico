# Accessibility Checklist

## WCAG AA Compliance

### Color & Contrast
- [ ] Text contrast ratio: 4.5:1 minimum (normal text)
- [ ] Large text contrast ratio: 3:1 minimum (18pt+ or 14pt bold+)
- [ ] UI components contrast: 3:1 minimum
- [ ] Color is not the only means of conveying information
- [ ] Focus indicators have sufficient contrast

### Typography
- [ ] Text can be resized up to 200% without loss of functionality
- [ ] Base font size is at least 16px
- [ ] Line height is at least 1.5 for body text
- [ ] Paragraph spacing is at least 1.5× font size
- [ ] No horizontal scrolling at 320px width (mobile)

### Keyboard Navigation
- [ ] All interactive elements are keyboard accessible
- [ ] Tab order is logical and intuitive
- [ ] Focus indicators are visible on all interactive elements
- [ ] No keyboard traps
- [ ] Skip to main content link available

### Screen Reader Compatibility
- [ ] Semantic HTML used throughout (headings, lists, landmarks)
- [ ] ARIA labels provided for icon-only buttons
- [ ] Form labels properly associated with inputs
- [ ] Error messages have role="alert"
- [ ] Dynamic content changes announced
- [ ] Images have meaningful alt text

### Forms
- [ ] All form fields have labels
- [ ] Required fields are marked
- [ ] Error messages are clear and helpful
- [ ] Success messages are announced
- [ ] Help text available where needed

### Images & Media
- [ ] Decorative images have empty alt="" attribute
- [ ] Informative images have descriptive alt text
- [ ] Complex images have long descriptions
- [ ] Videos have captions
- [ ] Audio has transcripts

### Responsive & Mobile
- [ ] Touch targets are at least 44x44px (iOS) or 48x48dp (Android)
- [ ] Content reflows without horizontal scrolling
- [ ] Orientation (portrait/landscape) both supported
- [ ] Zoom up to 200% without loss of functionality
- [ ] Content readable without zooming on mobile
